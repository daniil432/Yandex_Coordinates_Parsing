Программа предназначена для поиска возможных маршрутов между точками, заданными в виде координат.
Все необходимые для работы пакеты указаны в requirements.txt.
При первом запуске программы в виртуальном окружении она может вместо рассчётов начать устанавливать chromium,
так как он нужен для создания браузерных сессий (это не точно, и верно либо для requests-html, который тут используется,
либо для selenium, которого в данном проекте нет, но ранее рассматривался, узнать можно только на практике).
Если такое произошло, то, возможно, программа после установки закончит свою работу, и её нужно будет перезапускать.
Путь до файла, с которого будут считывться данные нужно указывать в функции read_dataset().
В этой функции можно вручную корректировать то, какие столбцы и строки файла считывать.
Для выбора столбцов для считывания нужно менять параметр usecols=, для пропуска строчек, 
например пустых, указывается параметр skiprows=, число считываемых строчек можно указать параметром nrows=.
Программа в ходе сохранения данных может выдать ошибку, если размерности матриц по какой-то 
причине не совпадут. Если такое произошло, то все полученные данные будут сохранены 
на отдельные листы, чтобы избежать потерь.
Во время работы программа может писать предупреждения вида "Удалённый хост принудительно разорвал существующее подключение",
такие сообщерния норма, это лишь показатель того, что сессия с одним из запросов по сформированной ссылке была закрыта.
Чтобы не возникало проблем со считыванием файлов и записью данных в файлы, 
рекомендуется не открывать считываемый файл перед запуском программы и 
не открывать файл, в который данные будут помещены, если он уже существует и 
данные в него будут перезаписываться повторно. Это рекомендовано потому, что у python
могут возникать проблемы с доступом к этим файлам, если они уже кем-то используются, например excel-ем.
